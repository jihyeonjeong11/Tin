# Retrospective (2026-03-29)

## 이 프로젝트의 진짜 목적

수익화 SaaS가 아니라:

1. **기술적 배움** — 요즘 SaaS 스택(Next.js + Express + Prisma + 모노레포) 직접 경험
2. **에이전틱 코딩 경험 및 개선방안 도출** — AI와 협업하는 워크플로우 검증
3. **재사용 가능한 SaaS 템플릿** — 다음 프로젝트에 복붙할 수 있는 기반

## 테스트에 대한 결정

- TDD(코드 전에 테스트 작성) 방식이 아닌, **기존 코드에 테스트를 붙이는** 접근
- 목표는 전체 커버리지가 아니라 **제네릭한 패턴 예시**
- 템플릿 작업 단계에서 작성 — 도메인 제거 후에 써야 Tin 특화가 안 됨
